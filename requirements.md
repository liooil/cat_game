# 像素猫猫盘文玩 - 游戏需求文档

## 1. 核心玩法
*   **放置与养成**：玩家初始资金 150 金币，可以领养猫咪、购买文玩手串。
*   **盘包浆机制**：把手串戴在猫咪脖子上，或者玩家手动盘串。包浆程度持续且实时地在面板刷新（0%~100%）。
*   **经济循环**：低价买入手串 -> 盘包浆 -> 高价卖出赚差价 -> 购买更高级的手串、更多的猫咪、家具和房间主题。
*   **全局可见性（修复猫咪消失）**：所有的猫咪都伴随玩家存在，当你切换房间（比如从默认小屋到花园洋房）时，所有你领养过的猫咪都会显示在当前场景中（不再消失）。

## 2. 美术风格 (星露谷物语风)
*   **整体风格**：精美、圆润的像素艺术（Pixel Art）。所有物体（包括猫咪和手串）底部带有半透明的环境阴影，增强体积感。
*   **画面比例**：画布调整为 800 x 480 像素，减少高度占比，使右侧视图显得更协调。
*   **猫咪**：
    *   身体边缘处理得更加“毛茸茸（Fluffy）”。
    *   行走时平滑移动，尾巴左右微微摆动（不再上下乱跳）。
*   **场景 (背景生活化与现实时间)**：
    *   引入现实时间光照系统，增加天空颜色的变幻（清晨/白天/黄昏/夜晚），光线可透过窗户影响室内亮度。
    *   **初始小屋 (默认)**：带有生活气息，增加一扇能看外面景色的窗户（去除奇怪的斜光，改为纯净天空）。背景放置小沙发和电脑桌，猫咪可以在前面活动。
    *   **温馨小屋 (北欧风)**：暖色木质调。配有铺着桌布的餐桌、小花瓶、墙上挂画、落地台灯，窗户更大。
    *   **花园洋房 (欧式多场景)**：包含三个可切换的场景区域（一楼、二楼、花园）。
        *   一楼：欧式窗户（能看到花园）、欧式楼梯、沙发。
        *   二楼：钢琴、大书架、桌上的欧式钢笔和黑色墨水。
        *   花园：开满郁金香（花朵坐标固定，修复乱抖动的 Bug），中间有一个猫猫形状的园艺草垛。
*   **手串立体化**：每个串（无论初始还是包浆后）都必须有阴影效果和内发光，看起来像真实的珠子（而不是平面的圆片）。
*   **家具与猫咪互动**：
    *   纯手绘的星露谷风格大号猫爬架、宽大猫抓板和双色猫窝。
    *   当购买对应家具后，猫咪会产生新动作：跳上猫爬架并在平台上卧着、在猫抓板上抓挠、在猫窝里睡觉。

## 3. 游戏实体与数据
### 3.1 猫咪 (Cats)
*   **品种**：橘猫、奶牛猫、狸花猫、白猫、黑猫、暹罗猫、三花猫。
*   **属性**：心情值 (0-100，随时间缓慢下降)。心情越高，帮玩家盘手串的速度越快。
*   **互动**：鼠标精确点击猫咪可选中，冒出“♥”漂浮动画。

### 3.2 商店与道具 (Shop)
*   **食物**：三文鱼（大幅恢复心情并赋予双倍包浆Buff）、普通罐头。
*   **手串**：紫金鼠、库克、菩提根、猴头、星月。

## 4. 架构与 UI 优化要求
*   **代码模块化（重构）**：`main.js` 文件过长，必须将代码拆分成多个独立的模块存放于 `js/` 目录下（如 `data.js`, `state.js`, `cat.js`, `renderer.js`, `ui.js`, `main.js`）。
*   **防止按钮跳动**：UI 面板（特别是左侧的猫咪信息区）必须设置固定高度或通过 `visibility` 来隐藏内容，杜绝文本变化或折叠展开导致左侧按钮集体跳动的 Bug。
*   **修复戴手串 Bug**：严格检查 `btn-give-cat` 按钮逻辑，确保选中的手串被正确赋值给选中的猫咪并从背包扣除。
