# 像素猫猫盘文玩 - 游戏需求文档

## 1. 核心玩法
*   **放置与养成**：玩家初始资金 150 金币，可以领养猫咪、购买文玩手串。
*   **盘包浆机制**：把手串戴在猫咪脖子上，或者玩家手动盘串。包浆程度（0%~100%）上升会增加手串的估值。
*   **经济循环**：低价买入手串 -> 盘包浆 -> 高价卖出赚差价 -> 购买更高级的手串、更多的猫咪、家具和房间主题。

## 2. 美术风格 (星露谷物语风)
*   **整体风格**：精美、圆润的像素艺术（Pixel Art）。带有阴影、体积感和环境细节。
*   **猫咪**：身体带有圆角，更立体的四肢和耳朵，有睡眠和走动的轻微摇摆动画，大眼睛（星露谷风格），底部带阴影。
*   **场景 (背景生活化与现实时间)**：
    *   引入现实时间光照系统，窗外的景色和透进屋内的阳光亮度随现实时间（早中晚）改变。
    *   **初始小屋 (默认)**：带有生活气息，增加一扇能看外面景色的窗户。背景放置小沙发和电脑桌，猫咪可以在前面活动，也可以跳到沙发或电脑桌上。
    *   **温馨小屋 (北欧风)**：暖色木质调。配有铺着桌布的餐桌、小花瓶、墙上挂画、落地台灯，并且窗户更大。
    *   **花园洋房 (欧式多场景)**：包含三个可切换的场景区域（一楼、二楼、花园），所有场景都可以领养猫和盘串。
        *   一楼：欧式窗户（能看到花园）、欧式楼梯、沙发。
        *   二楼：钢琴、书架、桌上的欧式钢笔和墨水。
        *   花园：开满郁金香，中间有一个猫猫形状的园艺草垛。

*   **家具与猫咪互动 (Furniture & Interaction)**：
    *   家具不再是简陋的 Emoji，而是用 Canvas 绘制出来的真正的像素风猫爬架、猫抓板和猫窝。
    *   **猫抓板需设计得更大**。
    *   当购买对应家具后，猫咪会产生新动作：
        *   跳上猫爬架并在平台上卧着。
        *   在猫抓板上抓挠。
        *   在猫窝里睡觉。

## 3. 游戏实体与数据
### 3.1 猫咪 (Cats)
*   **品种**：橘猫、奶牛猫、狸花猫、白猫、黑猫、暹罗猫、三花猫。
*   **外观优化**：身体边缘处理得更“毛茸茸（Fluffy）”一些，而不是僵硬的矩形。
*   **动画优化**：行走时尾巴不应该上下动，改为**左右微微摆动**或不动。
*   **状态**：增加与环境互动的状态，如：sit, walk_left, walk_right, sleep, play, on_sofa, on_desk, on_cattree, scratching, in_bed。
*   **属性**：心情值 (0-100，随时间缓慢下降)。心情越高，帮玩家盘手串的速度越快。
*   **互动**：鼠标点击猫咪可选中，并增加 5 点心情，头顶冒出“♥”漂浮动画。选中猫咪后，左侧面板中的**包浆程度数值应当实时、持续地变化显示**（不能只在点击时更新）。

### 3.2 手串 (Items)
*   **种类**：紫金鼠(10金)、库克(30金)、菩提根(50金，多宝渐变色)、猴头(100金)、星月(150金)。
*   **立体感优化**：每个串（无论初始还是包浆后）都必须添加阴影效果（Drop Shadow / Bevel），使其看起来具有立体感，而不是一个平面圆点。
*   **盘玩表现**：随着包浆程度增加，颜色向着各自的“包浆色”平滑过渡，并且产生白色高光。多宝菩提根会变得更加深沉油润。猴头和星月有独特的表面纹理。

### 3.3 商店与道具 (Shop)
*   **家具**：
    *   猫爬架 (200金)：+0.1/s 全局包浆速度。
    *   猫抓板 (150金)：+0.05/s 全局包浆速度。
    *   猫窝 (100金)：+0.02/s 全局包浆速度。
*   **食物**：
    *   三文鱼 (20金)：特殊道具。喂给选中的猫，大幅恢复心情 (+50)，且赋予 30秒 的“盘串速度翻倍”发光特效Buff。
    *   普通罐头 (5金，快捷面板)：恢复所有猫咪 20 点心情。
*   **房间主题**：
    *   温馨小屋 (500金)：暖色木质调。
    *   花园洋房 (1000金)：清新绿色调。

## 4. 界面与交互 (UI/UX)
*   **左侧面板**：资金显示、选中猫咪状态（心情、佩戴手串、**包浆进度持续刷新**、估值）、快捷操作（喂罐头、喂三文鱼、逗猫、穿戴/取下手串）。
*   **下方背包**：下拉框选择拥有的手串，可在此出售或长按手动盘玩。
*   **右侧渲染区**：游戏主画面。如果处在花园洋房，可以显示楼层切换按钮。
*   **操作修正要求**：
    *   **【极其重要】修复右侧Canvas区域无任何元素显示（渲染崩溃）的Bug**。必须确保 `gameLoop` 能稳定启动、持续调用 `requestAnimationFrame`，且在所有绘制函数（尤其是 `drawBead`、`drawFurniture`、`Cat.draw`）中绝对不能出现抛出异常的语法错误或 undefined/null 访问，这会导致整个 Canvas 停止渲染变为空白。
    *   修复 Tab 切换后时间差(dt)过大导致猫咪飞出屏幕消失的Bug（需限制单帧最大时间差）。
    *   确保 Canvas 坐标与鼠标点击坐标精确映射，修复无法选中猫咪的Bug。
    *   确保所有漂浮文字动画（如爱心、加成提示）能正常在主循环中渲染。
